####　创建原则
1.单一职责,一个组件只干一件事,如果组件变得复杂,那么就要拆分成小组件
2. DRY原则,能计算的状态就不要存到state中,组件尽量无状态,数据通过props获取.

### 组件的抽离

#### 为什么抽离组件。
1. 页面ui结构复杂，当个使用单个组件时，会使得该组件代码长度过大，降低代码的可阅读性，不便于维护。
2. 复杂 的 页面ui通常会具有很多相似的dom结构，使用单个组件构建页面ui会产生很多重复代码，不够优雅。
3. 不利于组件的复用
